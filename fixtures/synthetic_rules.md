# Sample Working Rules

These are fully synthetic rules used for tests and the demo fixture.

## Git hygiene

- Never use `--no-verify` when committing or pushing.
- Do not use `git push --force` against a shared branch.
- Use conventional commit messages.
- Run the test suite before committing.

## Database

- Never use the `--no-ssl-verify` flag when connecting to the database.

## Output style

- No emojis in assistant output.
- Prefer short, direct answers.

## Behavioral

- Act like a senior engineer.
- Prefer simplicity over cleverness.
